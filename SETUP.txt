======================================================
  BLITZ DASHBOARD — SETUP & DEPLOYMENT GUIDE
======================================================

-----------------------------------------
OPTION A: RUN LOCALLY (Recommended first)
-----------------------------------------

1. Install Python 3.10+ if not already installed
   → https://www.python.org/downloads/

2. Open Terminal / Command Prompt, navigate to this folder:
   cd "Blitz Dashboard"

3. Install dependencies:
   pip install -r requirements.txt

4. Run the app:
   streamlit run Home.py

5. Your browser will open at: http://localhost:8501
   Share on your local network at: http://YOUR_IP:8501

--------------------------------------------------
OPTION B: DEPLOY TO STREAMLIT COMMUNITY CLOUD (Free, shareable URL)
--------------------------------------------------

1. Create a free GitHub account at https://github.com
2. Create a new repository (e.g. "blitz-dashboard"), upload all files in this folder
3. Go to https://share.streamlit.io and sign in with GitHub
4. Click "New app" → select your repo → set Main file = Home.py
5. Click Deploy — your app will be live at:
   https://your-repo-name.streamlit.app

Anyone with the link can access it. No login required by default.

--------------------------------------------------
WEEKLY UPDATE WORKFLOW
--------------------------------------------------

1. Update your Excel file as usual (add new week's data to Raw Data Source)
2. Open the Streamlit app in your browser
3. Click "Browse files" on the Home page, select your Excel file
4. All 7 pages update instantly — no pivot refresh needed

## Public Deployment

To deploy this dashboard to Streamlit Community Cloud with auto-updating data:

1. **Create GitHub repo & push code**
   - Create a new GitHub repo (e.g. "blitz-dashboard")
   - Clone locally, copy all files from this folder
   - Push to main branch: git push -u origin main

2. **Create data/ folder with .gitkeep**
   - mkdir data && touch data/.gitkeep
   - git add data/.gitkeep && git commit -m "Add data folder" && git push
   - (Files uploaded via Updater page will be committed here)

3. **Deploy to Streamlit Community Cloud**
   - Go to https://share.streamlit.io
   - Sign in with GitHub
   - Click "New app"
   - Select your repo, set Main file to "Home.py"
   - Click "Deploy"
   - Your app goes live at: https://your-username-blitz-dashboard.streamlit.app

4. **Add secrets for Admin Updater**
   - In Streamlit Cloud app settings → Secrets
   - Add the following TOML:

     [github_token]
     # Generate at https://github.com/settings/tokens
     # Scope: repo (full control of private repositories)
     # Example: ghp_1234567890abcdefghijklmnopqrstuvwxyz

     [github_repo]
     # Format: your-username/your-repo-name

     [github_branch]
     # Default: "main"

     [admin_password]
     # Your strong password for the Updater page
     # Example: MyStr0ng!P@ssw0rd

   Full example:
   ```
   admin_password = "your-strong-password-here"
   github_token = "ghp_..."
   github_repo = "your-username/blitz-dashboard"
   github_branch = "main"
   ```

5. **Share with your team**
   - Send the public URL to your team
   - Only you (with admin password) can upload data via Updater page
   - Everyone can view all pages

**How it works:**
- First user to visit Home page → uploads data file → auto-saves to data/ folder
- OR: Admin visits Updater page → uploads file → commits to GitHub → app auto-redeploys in ~30s
- All users see updated data in next page load (within 30s)

**Weekly update workflow:**
1. Admin logs into Updater page (🔐 Updater in sidebar)
2. Upload new Delivery .xlsx or Mobile Sellers .xlsx
3. Click "Publish" button
4. GitHub commit + Streamlit Cloud redeploy automatically
5. All users see fresh data in next page load
